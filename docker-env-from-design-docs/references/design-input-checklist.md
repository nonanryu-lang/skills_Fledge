# 設計入力チェックリスト

## 1. 入力ドキュメント確認

- 基本設計書が存在する
- 詳細設計書が存在する
- 改訂履歴（版数、更新日、変更理由）が追える
- 未確定事項（[TBD]）が識別できる

## 2. サービス境界確認

- サービス一覧（Web / API / Worker / Batch / DB / Cache / Queue）
- 各サービスの責務
- 同期・非同期の通信経路
- 外部サービス依存（SaaS、社内API、認証基盤）

## 3. 実行要件確認

- ランタイム種別とバージョン
- 必須システムパッケージ
- ビルド手順
- 起動コマンド
- ヘルスチェック方法

## 4. データ要件確認

- DB種別とバージョン
- スキーマ作成方法
- マイグレーション手順
- シードデータ有無
- 永続化対象（ボリューム）

## 5. 設定・機密情報確認

- 環境変数一覧
- 機密情報と非機密情報の分離
- ローカル用ダミー値提供可否
- 設定ファイルの優先順位

## 6. 運用要件確認

- ログ出力先とフォーマット
- 監視対象（死活、遅延、エラー率）
- リソース上限（CPU/メモリ）
- 再起動ポリシー
- 障害時の復旧手順
