# Docker環境検証チェックリスト

## 1. 構文・整合チェック

- `docker compose config` が成功する
- サービス名、ポート、ボリューム名の重複がない
- `.env.example` と compose のキーが一致する

## 2. 起動チェック

- `docker compose up --build -d` が成功する
- 主要サービスが `healthy` になる
- 依存順（db→api→web など）が成立する

## 3. 機能経路チェック

- Web画面にアクセスできる
- APIヘルスチェックが成功する
- DB接続が成立する
- 必須の外部接続設定が反映される

## 4. 初期化・データチェック

- マイグレーションが成功する
- 初期データ投入が成功する（必要な場合）
- コンテナ再起動後も永続データが保持される

## 5. 復旧・運用チェック

- サービス再起動手順が再現できる
- ボリューム再作成手順が明記されている
- ログ確認コマンドが提示されている
- トラブル時の切り分け手順が提示されている
